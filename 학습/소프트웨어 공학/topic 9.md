# Release Testing
- 시스템의 특정 릴리즈를 테스트
- 시스템 공급자에게 해당 시스템이 사용하기에 충분히 좋음을 확신시키게 됨
	- 명시된 기능을 전달하며 요구되는 성능 및 의존성을 달성하며 일상적인 사용에서 fail되지 않음을 보여주어야 함
- 보통의 경우 black-box 테스트, 테스트는 시스템 명세서를 통해 추출하게 됨

## 릴리즈 테스트 시스템 테스트
릴리즈 테스트는 시스템 테스트의 형태임
차이점
- 시스템 개발에 관여하지 않은 별도 팀에서 수행
- 개발팀 -> 버그
